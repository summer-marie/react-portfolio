# Project Documentation

This directory contains the planning, design, engineering, and implementation documentation for the portfolio.

These documents should be read in order before making significant architectural, design, or engineering decisions.

---

## Reading Order

### 00 — Project Vision

Defines the purpose, audience, goals, design principles, and overall direction of the portfolio.

---

### 01 — Visual Language

Documents the visual identity including typography, color palette, spacing, motion philosophy, and overall design aesthetic.

---

### 02 — Site Architecture

Defines the site's navigation, page hierarchy, content organization, and information architecture.

---

### 03 — Motion and Components

Documents reusable UI components, interaction patterns, animations, accessibility considerations, and component behavior.

---

### 04 — Build Plan

Provides the implementation roadmap, project phases, testing requirements, Git workflow, and Definition of Done.

---

### 05 — Engineering Standards

Defines implementation standards including responsiveness, accessibility, performance, coding conventions, component philosophy, and maintainability.

---

### 06 — Tech Stack

Documents the project's architectural decisions, tooling, dependencies, deployment strategy, and technology rationale.

---

## Related Documentation

The repository root also contains:

* **AGENTS.md** — Shared operating instructions for AI coding agents.
* **CLAUDE.md** — Claude-specific startup and workflow instructions.

---

## Updating Documentation

Documentation should evolve alongside the project.

Update documentation whenever architectural decisions, engineering standards, or implementation strategies change.

Minor implementation details that do not affect project direction generally do not require documentation updates.

These documents serve as the project's long-term source of truth.
